# 用户界面调整计划

## 1. 需求分析
- 删除用户中心下方多余的编辑个人信息按钮
- 在"我的"界面中增添"个人信息"一栏，点击可查看当前个人信息
- 新创建的个人信息组件也需要包含编辑功能
- 确保不显示多余字段

## 2. 实现方案

### 2.1 删除用户中心多余按钮
- **修改文件**：`UserCenter.vue`
- **操作**：删除`user-actions`区域中的"编辑个人信息"按钮
- **保留**：只保留用户信息部分的"编辑个人信息"按钮

### 2.2 在"我的"界面添加个人信息标签页
- **修改文件**：`UserMyPage.vue`
- **操作**：在现有标签页后添加"个人信息"标签页
- **组件选择**：直接使用现有的`UserProfile`组件

### 2.3 优化UserProfile组件（如果需要）
- **修改文件**：`UserProfile.vue`
- **操作**：确保组件适合在标签页中使用
- **注意**：不添加多余字段，只保留必要的编辑功能

## 3. 技术细节
- 使用Vue 3 Composition API
- 利用现有的authStore获取用户信息
- 保持与现有UI风格一致
- 确保不显示多余字段
- 直接复用现有组件，避免冗余开发

## 4. 预期效果
- 用户中心只显示一个"编辑个人信息"按钮
- "我的"界面包含三个标签页：我的训练计划、我的消息、个人信息
- 个人信息标签页显示当前用户的基本信息，并提供编辑功能
- 编辑功能与现有逻辑保持一致
- 不显示任何多余字段

## 5. 为什么直接使用现有组件
- `UserProfile.vue`已经包含了完整的个人信息编辑功能
- 直接复用可以节省开发时间，避免重复代码
- 确保编辑功能的一致性
- 可以通过修改组件参数（如果需要）来适应标签页环境
- 不需要创建新的组件，减少项目复杂度
